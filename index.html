LeanMate OOP Architecture

아래 구조는 LeanMate의 기능 클래스를 상속 관계로 나타낸 것입니다. 최상위 부모 클래스인 FitnessModule은 공통 run() 메서드를 정의하고, 자식 클래스들은 이 메서드를 오버라이딩하여 목표 계산, 음식 검색, 식단 기록, 운동 출석 기록처럼 서로 다른 결과를 출력합니다. Git 관리는 main 브랜치를 안정 버전으로 유지하고, 기능별 브랜치에서 작업한 뒤 병합하는 방식으로 진행했습니다.

상속 FitnessModule을 부모 클래스로 두고 목표, 음식 검색, 식단, 운동 출석 기능이 확장됩니다.
오버라이딩 자식 클래스는 같은 run() 메서드를 자신의 역할에 맞게 다시 정의합니다.
다형성 같은 run() 호출이 객체 타입에 따라 칼로리, 식단, 랭킹 계산으로 바뀝니다.

모든 기능 클래스는 같은 run() 메서드를 가지고 있지만, 각 클래스가 이를 오버라이딩하여 서로 다른 계산 결과를 출력합니다.

GoalPlan에서 목적별 목표 플랜으로 상속
FitnessModule.run():
부모 클래스의 공통 인터페이스입니다. 클릭한 자식 클래스가 같은 run() 메서드를 오버라이딩해 서로 다른 계산 결과를 출력합니다. 
const selectedModule = modules[className];
const result = selectedModule.run(context);
1. 목표 계산 `createGoalPlan(goalMode)`가 감량, 유지, 린매스업 중 알맞은 자식 클래스를 만들고 `run(profile)`로 하루 목표를 계산합니다.
2. 음식 검색과 식단 합산 `FoodSearchModule.run(query)`는 음식 Map을 조회하고, `MealRecord.run(target)`은 오늘 식단 배열의 총 섭취량을 계산합니다.
3. 운동 출석 랭킹 `AttendanceRecord.run(selectedDate)`는 날짜별 운동 출석 기록을 바탕으로 주간 랭킹과 연속 출석일을 계산합니다.
