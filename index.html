음식 데이터는 Map으로 빠르게 찾고, 오늘 식단은 배열로 누적하며, 출석 기록은 랭킹으로 정렬합니다. 아래 OOP 다이어그램의 클래스를 누르면 같은 run() 메서드가 서로 다른 방식으로 실행됩니다.

OOP 상속 구조와 다형성 인터랙션

부모 타입 FitnessRecord 배열에 서로 다른 자식 객체를 담고, 클릭 시 각자의 run() 오버라이딩 결과를 보여줍니다.

상속 extends
FitnessRecord.run()
부모 클래스는 실행 규칙만 정의합니다. 자식 클래스를 클릭해 다형성 결과를 확인하세요. 
const records = [
  new MealRecord("닭가슴살 식단"),
  new WorkoutCheckIn("오늘 운동"),
  new GroupChallenge("헬스장 경쟁")
];

records.forEach(record => record.run());

그룹 운동 출석

같은 목표를 가진 사람끼리 오늘 운동 여부를 체크하고 주간 출석 횟수를 비교합니다.

주간 랭킹

출석 횟수를 기준으로 높은 순서대로 정렬합니다.

사용한 자료구조

Map 음식 이름을 key로 저장해 영양성분을 빠르게 조회합니다.
Array 오늘 먹은 식단 목록과 그룹원 목록을 순서대로 관리합니다.
Stack 마지막 식단 추가 기록을 되돌리는 undo 기능에 사용합니다.
Sort 주간 출석 횟수 기반 랭킹을 계산합니다.
