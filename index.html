LeanMate OOP Architecture

LeanMate의 핵심 기능은 상속 구조로 설계되어 있으며, 각 클래스 카드를 클릭하면 오버라이딩된 run() 메서드가 다형성에 의해 서로 다른 계산 결과를 출력합니다.

상속 FitnessModule을 부모 클래스로 두고 목표, 식단, 출석 기능이 확장됩니다.
오버라이딩 자식 클래스는 같은 run() 메서드를 자신의 역할에 맞게 다시 정의합니다.
다형성 같은 run() 호출이 객체 타입에 따라 칼로리, 식단, 랭킹 계산으로 바뀝니다.
상속 extends
GoalPlan을 상속한 목적별 플랜
FitnessModule.run():
부모 클래스의 공통 인터페이스입니다. 실제 기능은 자식 클래스가 오버라이딩하며, 같은 run() 메서드 호출이 다형성에 의해 서로 다른 결과를 만듭니다. 
// FitnessModule.run(): 공통 인터페이스
const plan = createGoalPlan(goalMode);
const target = plan.run(profile);

const mealRecord = new MealRecord(todayMeals);
const mealSummary = mealRecord.run(target);

const attendance = new AttendanceRecord(members);
const ranking = attendance.run(selectedDate);
